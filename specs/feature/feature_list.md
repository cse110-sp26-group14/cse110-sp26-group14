User Features

1. **Login System**  
   1. Users can sign up, log in, and access personalized sprint information.  
   2. User profiles store availability, assigned tasks, reports, and sprint activity.

2. **Sprint Calendar**  
   1. Displays the current month and active sprint schedule.  
   2. Active sprint weeks are visually highlighted.  
   3. Calendar displays:  
      1. Sprint deadlines  
      2. Meeting dates and times  
      3. Meeting format (online, in-person, hybrid)  
      4. Zoom links or locations  
   4. Interacting with a sprint deadline displays sprint-specific information for the user.

3. **Project Information Dashboard**  
   1. Displays the current completion status of the project.  
   2. Shows priority tasks and current blockers.  
   3. Displays team activity and sprint progress.  
   4. Dashboard content updates dynamically depending on the selected sprint.

4. **Async Check-In & Reporting System**  
   1. Users receive reminders to submit short daily progress updates asynchronously.  
   2. Users can submit:  
      1. Completion status  
      2. Current progress  
      3. Blockers or issues  
      4. Additional comments or notes  
   3. Reports are saved and attached to the current sprint.

5. **AI Team Summary System**  
   1. AI summarizes daily team reports and blockers.  
   2. Summaries help synchronize team progress without requiring synchronous meetings.  
   3. Original user reports remain accessible for transparency.

6. **AI Sprint Task Suggestion (Admin Exclusive)**  
   1. Managers can describe project goals or deadlines.  
   2. AI generates suggested sprint tasks based on project requirements and reported issues.

7. **Issue / Blocker Tracking**  
   1. Users can create issue reports at any time.  
   2. Issues can generate new tasks or update sprint priorities.  
   3. Issues are displayed in a dedicated Issues List page.

8. **Activity Log / Backlog**  
   1. Stores previous reports, sprint updates, AI summaries, and blocker history.  
   2. Supports filtering by sprint, task, user, or priority.  
   3. Allows teams to review project history without searching through chat applications.

9. **Availability Tracking**  
   1. Users can submit availability updates.  
   2. Availability information can help suggest meeting times and sprint planning adjustments.  
   3. Green icon next to username indicating users are active/logged in/available.
