# Title: Calendar Implementation

**Status**: Approved

**Author**: Kaitlyn Nguy

**Context**: 
    One of the features decided upon for deisgning the SitRep was the inclusion of organizing tasks and meeting dates across the team to see with easy access. The our decided to communicate a part of that would be through a calendar feature that allowed for meetings and task due dates to be visible on a sprint to sprint basis. 

**Considered Options**: 
1. Google Calendar
   1. Pros: 
   2. Cons: 
2. Outlook Calendar

**Chosen Decision**: Google Calendar

**Rationale**: 
Widley used among a large amount of individuals. Google Calendar has an implemented way of tracking individual availability, thus creating new events based on changes to that availability would be much easier to implement. On Events, Google Meets are able to be created to make setting up calls or communication much easier. The API itself is much easier to implement and understand as many functions are already implemented to sign in, be accessed, and be used. 

**Consequences**: 
Not everyone uses Google Calendar which can limit the amount of users that use our software. Google API's can intrinsically have risks to security due to the amount of people who use such APIs, potentially causing risks for viruses for example. Only one person is able to fully manage the calendar as the API is tied to one specific key.
